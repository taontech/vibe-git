实现 Node.js 侧的 Agent Monitor 进程管理，使其随 GMC Web 启动、复用和退出。

实施范围：
- 依赖“整理并打包 Agent Monitor 服务”任务确定稳定的服务路径和启动参数。
- 可修改 `cli/bin/gmc.js`；优先新增 `cli/lib/agent-monitor.js` 封装探活、启动、超时、日志和停止逻辑。
- GMC Web 启动时先探测配置端口上的 `/health`；已有兼容实例时复用，否则启动随包提供的服务并等待其就绪。
- 支持通过环境变量覆盖端口、Python 路径或禁用监控；默认端口保持 `8898`。
- `gmc web --quit`、前台退出及重启流程应只清理当前 GMC 启动的子进程，不得终止用户独立运行的 Agent Monitor。
- Python、依赖缺失或端口冲突时，GMC Web 仍可启动，并提供可诊断的降级状态。

验收标准：
- 前台、后台、重启和退出模式下均不会产生重复监控实例或遗留由 GMC 创建的进程。
- 已独立运行的兼容监控服务能被安全复用。
- 监控启动失败不会导致 Git、任务等现有 Web 功能不可用。
- `npm test` 通过。
