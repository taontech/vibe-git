在 Web 仪表盘中实现完整的多主题能力。主要修改现有文件 `cli/lib/web.js`；由于页面、样式和设置逻辑均内联于该文件，为避免合并冲突应在同一任务中完成。

实现范围：
- 建立统一的主题配置机制，覆盖页面背景、卡片、边框、按钮、输入框、状态提示等组件颜色，以及标题、正文、弱化文本等字体颜色。
- 内置至少三种视觉上可明确区分的主题，并为每种主题提供完整配置；现有界面风格可作为其中一种主题。
- 在设置页增加主题色/主题选择控件，展示可选主题并明确标识当前主题。
- 用户选择后立即将主题应用到整个 Web 仪表盘，无需刷新页面。
- 持久化用户选择，使刷新页面或重新打开仪表盘后仍使用上次选中的主题；未保存选择或保存值无效时安全回退到默认主题。
- 确保主题切换不影响现有设置项、交互逻辑、响应式布局和安全相关功能。

验收标准：
- 设置页可选择至少三种主题。
- 每种主题均会同步改变主要组件颜色和字体颜色，而不只是单一强调色。
- 切换主题后所有已有页面区域保持文字清晰、状态可辨识、控件可正常操作。
- 刷新及重新打开页面后主题选择保持不变。
- 无效或缺失的主题配置不会导致页面报错，并回退到默认主题。
- 在 `cli/` 目录运行 `npm test`，所有 JavaScript 文件通过语法检查。
