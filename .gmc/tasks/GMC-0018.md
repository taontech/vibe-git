为 GMC Web 增加同源监控接口，并在任务页各 Agent 区域展示对应的设备运行信息。

实施范围：
- 依赖“随 GMC Web 管理监控服务生命周期”提供监控地址和健康状态。
- 可修改 `cli/lib/web.js`；该任务集中完成该文件中的路由、页面样式、文案和浏览器逻辑，以避免拆分造成大量合并冲突。
- 新增受现有 GMC 鉴权和外部访问策略保护的只读 API，由服务端请求 Agent Monitor 的 `/agents` 或 `/status`，设置短超时、校验 JSON，并返回明确的 unavailable/error 状态；浏览器不得直接跨端口访问 Python 服务。
- 建立任务列与监控 ID 的映射：`codex` 对应 `codex-cli`/`codex-app`，`claude` 对应 `claude-code`，`antigravity` 对应 `antigravity`；多个进程或多个 Codex 类型需正确聚合或分别展示。
- 在对应 Agent 列头或 Agent 信息框显示运行状态、进程数、CPU、内存和最长运行时长；状态颜色和动画应基于监控返回的真实状态，而不是仅根据列中是否存在任务判断。
- 进入任务页、手动刷新及页面保持打开时定期更新监控数据；离开任务页或页面卸载时停止轮询，避免重复定时器。
- 为加载中、未运行、监控不可用、超时和未知 Agent 提供不阻塞任务操作的降级展示，并补齐项目现有语言表中的相关文案。

验收标准：
- 模拟 `working`、`idle`、无进程和服务离线响应时，各 Agent 信息框显示正确且任务看板仍可正常拖放、编辑和删除任务。
- Agent Monitor 不可达或返回畸形数据时，同源 API 有界限地失败且不泄露内部堆栈。
- 非 Agent 列不显示无关监控信息；Codex CLI 与 Codex App 的数据不会遗漏。
- `npm test` 通过，并通过浏览器验证窄屏与宽屏布局。
