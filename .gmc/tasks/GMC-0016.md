将 `agent-monitor` 服务整理为可随 npm 包发布的运行时资源。

实施范围：
- 可修改 `agent-monitor/server.py`、`agent-monitor/start.sh`、`agent-monitor/requirements.txt`、`cli/package.json`；可在 `cli/` 下新增打包后的服务目录或安装脚本。
- 排除 `.venv`、`__pycache__` 等本机生成内容，确保 npm 包只包含服务源码和依赖声明。
- 修正服务中写死的 `8898`：配对响应、二维码、UDP beacon、Bonjour 信息均应使用实际启动端口。
- 保留 `API_DOCUMENTATION.md` 定义的 REST、WebSocket、UDP 和 Bonjour 行为。

验收标准：
- `npm pack --dry-run` 能看到运行 Agent Monitor 所需的源码和依赖文件，但不包含虚拟环境及缓存。
- 使用非默认端口启动时，`GET /health`、`GET /agents` 可访问，所有对外发布的端口信息均与实际端口一致。
- 现有 Agent Monitor Python 测试通过。
